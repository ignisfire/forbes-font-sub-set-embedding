Font sub-setting to avert CLS for forbes.com article headings

The technique strips non-essential characters and functionality from a font file, leaving only those needed to display a normal forbes.com article title, and embedding the resulting font file inline in the CSS as base64 encoded string. Stripping reduces the size by at least 50%. For our article pages this will increase the (un-compressed) size of the html by 15kb (estimated 8kb actual traffic). This demo is best viewed in full-screen, and the external font loading is intentionally delayed by 2 seconds to mimic a slow internet connection.
